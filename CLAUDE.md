# Проект: изометрический градостроительный симулятор для Steam (TS + React + PixiJS + Tauri)

Полный план и дизайн систем — в PLAN.md. Сверяйся с ним.

## Железные правила архитектуры

- `src/sim/` не импортирует ничего из `render/`, `ui/`, pixi, react, tauri. Только чистый TS.
- Платформенные API (файлы, Steamworks) — только через интерфейс `Platform`,
  чтобы dev-режим работал в браузере без Tauri.
- Любое изменение GameState — только через команды (`sim/commands.ts`) с валидацией.
- Симуляция детерминирована: никаких `Date.now()`/`Math.random()` внутри `sim/` —
  время приходит тиками, случайность через seeded RNG из состояния.
- Весь игровой контент — данные в `sim/content/`, код их интерпретирует.
- Числа баланса — только в `content/balance.ts`.
- Все бонусы/штрафы (великие здания, эдикты, счастье, технологии) — только
  через систему модификаторов `sim/systems/modifiers.ts`, без точечных `if`.
- Монетизация: никакой премиум-валюты, платных ускорений или IAP-заглушек.
  Контент — только разделяемые data-паки (эпохи/косметика) под DLC или бесплатно.
- Все строки UI — через i18n (`src/i18n/en.json`, `ru.json`), никаких строк в JSX.
- Информация никогда не кодируется только цветом: каждый ресурс = иконка + цвет,
  статусы зданий и шкал — символами/формой в дополнение к цвету.

## Качество

- TypeScript strict, без `any`. Новые системы симуляции — с юнит-тестами (Vitest).
- После изменений: `npm run lint && npm test`.
- Спрайты не хардкодить: рендер берёт текстуры из атласа по `building.defId`.

## Стиль работы

- Перед большой фичей — краткий план файлов и типов, потом код.
- Не переписывай работающие системы без запроса, расширяй.
